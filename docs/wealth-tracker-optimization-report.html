Agentic Coding 标准对照

当前状态

  • 无契约文档 — 无 acceptance criteria，改完不知道"做到哪算完"
  • 单体巨石 — 209KB 单文件，逻辑混在 HTML 里，无法做 surgical diff
  • 12 个函数重复定义 — 最后定义覆盖前者，增加维护心智负担
  • 无测试证据 — 修完 bug 无 before/after 对比
  • 无交付文档 — 新人接手需通读全部代码

Agentic Coding 标准

  • Code Contract — 每次修改前锁定目标+验收+禁区
  • PACT Loop — Problem → Acceptance → Change → Trace
  • Surgical Diff — 一次目标一个聚焦变更
  • Prove Fix — before/after 证据
  • Handoff Packet — 改了什么、影响范围、回滚路径
核心矛盾：Agentic Coding 要求"一个目标一个聚焦变更"，但当前架构下任何 UI 改动都必须在 209KB 单文件中操作，天然违反 surgical diff 原则。建议先 P0-P2 功能迭代，P3 做模块拆分。

12 个具体问题

#优先级问题位置影响
1P0 轮动操作跨 Tab 碎片化 — 期权信号在控制台，期权记录在期权tab，策略矩阵在期权tab底部。用户执行一次轮动需要切换 2-3 个 tab。 tab-console + tab-option 每次操作多 3-5 次点击
2P0 到期预警不可见 — checkRollAlerts() 只在期权 tab 内渲染，仪表板看不到即将到期的 CALL。轮动策略核心决策信息被隐藏。 checkRollAlerts (L7108) 可能错过 roll 窗口
3P0 轮动建议被淹没 — buildActionLine() 输出的 "VGT:2CALL@xxx | SMH:1PUT@xxx" 是核心决策，但只用一行文字在"策略参考"区展示。无视觉层次、无操作入口。 buildActionLine (L7049) 最核心的 AI 建议被忽略
4P0 VIX 信号冗余重复 — VIX + CALL SWITCH + EST. INCOME 卡片在控制台和期权 tab 各出现一次，ID 不同但内容相同。 tab-console + tab-option 信息冗余，维护成本高
5P1 Tab 顺序不匹配使用频率 — 规划（低频）排在第 6，日志（低频）排在第 5。高频的期权/轮动操作被低频页面隔开。 全 Tab 导航 日常使用多 30% 的切换
6P1 交易录入和期权录入割裂 — 买卖股票在控制台，期权记录在期权 tab。同属"交易"操作，表单分布在两个 tab。 tab-console + tab-option 操作路径割裂
7P1 持仓明细藏在数据 tab — 持仓列表 (holdBody) 在 tab-data，持仓总览 (metrics + donut) 在 tab-holding。想看完整持仓需要跳两个 tab。 tab-holding + tab-data 持仓信息不完整
8P2 侧边栏 DCA 设置不醒目 — 每月定投金额和分配比例在侧边栏底部小 input 里，视觉权重低。 sidebar (L2883) 设置不便
9P2 规划 tab 无当前进度关联 — 20 年路线图和退出策略显示硬编码数字，没有联动当前 portfolio 实际值。 tab-plan 策略与现状脱节
10P2 日志 tab 信息密度低 — 活动日志 + 纪律打卡 + 热力图占用独立 tab，日常打开率极低。 tab-log 浪费 tab 位
11P3 移动端底部导航 6 项过密 — 6 个按钮在 375px 下每个约 55px，触摸目标偏小。 bottom-bar 误触率高
12P3 再平衡与交易录入混在同一页 — 年度低频操作不应与每日交易入口同级。锁定状态不透明。 tab-console 低频占用高频位

优化方案

P0 — 立即修复（影响轮动策略核心体验）

1. 新建「轮动」Tab，合并控制台 + 期权

将 tab-console 和 tab-option 合并为 tab-wheel，按轮动工作流自上而下排列：

区块内容来源
市场信号VIX 仪表、VGT/SMH 现价、趋势指示（涨/横/跌）原控制台信号卡
活跃持仓 & 到期预警当期 CALL/PUT 列表 + 到期倒计时，ITM 高亮红色 + Roll 按钮原期权持仓清单 + rollAlert 上移
轮动建议buildActionLine() 输出做成卡片：VGT 写 2 CALL @ $xxx (+8% OTM)，附带一键填入交易表单原策略参考区，重新设计
交易录入股票买卖 + 期权记录统一表单，类型选择切换原控制台 + 期权表单合并
OTM 策略矩阵原 renderMatrix() 的 CALL/PUT 表格，当前高亮原策略参考底部
盈亏汇总本月权利金 + 累计 + 已实现（3 metric cards）原期权盈亏汇总
收益：用户在一个页面完成「看信号 → 看建议 → 执行交易」的完整轮动闭环，从跨 2-3 个 tab 减少到 1 个页面内垂直滚动。

2. 到期预警提升到全局 Banner

将 checkRollAlerts() 输出从期权 tab 内部移到仪表板顶部（跨 tab 可见的固定 banner）：

  • 在 updateAll() 中调用 checkRollAlerts() 并渲染到 #tab-holding 顶部
  • Banner 样式：橙色底色 + 到期天数 + 一键 Roll 按钮
  • 无到期时自动隐藏，不占空间

3. 轮动建议卡片化，支持一键填表

将 buildActionLine() 文本输出改为可交互卡片：

  • 每行建议 = 一张卡片：标的 + 类型(CALL/PUT) + 行权价 + OTM% + 建议张数
  • 点击卡片自动填入下方交易表单（预填 symbol/type/strike/shares）
  • CALL 绿色调 · PUT 蓝色调 · 预警红色调
  • 附注原因文字（如 "趋势上涨，远PUT +12% OTM"）

4. 消除 VIX 信号重复

合并后只有一个 VIX 信号区域。保留控制台版本的数据源（updateOptionSignals()），移除期权 tab 中的重复渲染。

P1 — 高优先级（改善日常使用习惯）

5. 重排 Tab 顺序：仪表板 → 轮动 → 持仓 → 规划 → 数据

新顺序Tab 名频率合并自
1仪表板每次打开原 holding（不变）
2轮动每日/每周console + option 合并
3持仓每次操作后原 data 中的 holdBody 上移
4规划每月/每季原 plan（不变）
5数据按需查阅原 data + log 合并
移动端底部导航从 6 减到 5：仪表板 · 轮动 · 持仓 · 规划 · 数据。375px 下每个约 68px，触摸目标舒适。

6. 统一交易录入表单

在合并后的轮动 tab 中，交易表单支持模式切换：股票买卖 | 期权记录。股票显示价格+数量，期权显示行权价+权利金+到期日+张数。按需显示对应字段。

7. 持仓明细上移到持仓 Tab

将 #holdBody 从 tab-data 移到 tab-holding 下部。tab-holding 变为：上=指标总览 + 中=目标进度+回撤 + 下=持仓明细表+持仓分布donut。

P2 — 体验增强

8. 侧边栏增加轮动收入卡片

在实时价格下方新增：本月权利金收入（绿色）、活跃合约数、最近到期日（即将到期高亮）。

9. 规划 Tab 联动实时数据

  • 20 年路线图进度条联动当前实际 portfolio value
  • 退出策略显示当前实际年龄进度
  • 提款模拟默认值联动实际总资产

10. 日志 Tab 降级为数据 Tab 子区块

将 tab-log 内容（活动日志 + 纪律打卡 + 月度热力图）作为 tab-data 底部的一个 collapsible card。删除独立的日志 tab。

P3 — 长期改进（架构层面）

11 & 12. 模块拆分 + 再平衡独立化

  • 模块拆分：将 JS 核心逻辑抽取到独立 .js 文件（portfolio.js、options.js、ui.js），HTML 只保留 UI 渲染。这是 Agentic Coding "surgical diff" 的前提
  • 再平衡独立：将再平衡面板从轮动 tab 移到规划 tab，与 DCA 调整、目标重设同属"定期策略调整"
  • 消除重复函数：清理 12 个重复的函数定义
