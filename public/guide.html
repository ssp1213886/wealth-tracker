投资策略

本工具围绕「永久核心仓 + Covered Call 增强」策略设计，仅卖出 CALL、不卖正股（不做空、不卖标的）：

  • 核心仓永不动：VGT / SMH / BTC 按月定投，侧边栏可调节分配比例
  • Covered Call 收租：持有正股卖出虚值 CALL，收获权利金
  • 被动行权 T+1 买回：若 CALL 被行权（股价涨超行权价），T+1 手动买回正股，保持仓位不变
  • 权利金复投：权利金收入按核心仓比例继续买入标的
  • 回撤应对：回撤时通过定投加仓降低成本，不增额外风险
  • 铁律：永不满仓卖出 CALL，始终保留至少 1 张不卖
核心铁律：
① 不卖标的 — 只卖 CALL、不卖正股，永不做空
② 永不满仓 — 始终保留 1 张不卖
③ 被动行权 — 被行权后 T+1 买回，不提前平仓
④ 权利金复投 — 权利金按比例持续买入标的

1. 页面结构

本网站包含 5 个页面，通过顶部 Tab（桌面端）和底部导航（移动端）切换。切换时提供触觉反馈（移动端短暂震动）：

📊 仪表盘

  • 总资产指标：账户现金、持仓市值、浮动盈亏、总盈亏
  • 📋 本月定投目标：VGT/SMH/BTC 动态金额 + 完成/待执行状态
  • 🗺 20年旅程：时间进度条（可配置起始日期和年龄）
  • 💎 目标进度：可配置财务目标进度条（带 shimmer 动画）
  • 📉 高点回撤：各标的数据历史交易峰值跌幅 + 回撤进度条
  • 📈 仓位盈亏：按标的汇总盈亏
  • 🍩 持仓分布：环形图 + 现金池标注
  • 💎 Covered Call：本月 CC 收入卡片
  • 期权持仓：活跃期权合约摘要（标的/类型/行权价/张数/到期日）

🕹️ 操作台

  • 买卖录入：手动录入交易（日期、标的、数量、价格），支持 VGT/SMH/BTC/SGOV
  • ⚖️ 年度再平衡：仅 12月31日解锁执行，其他日期可预览计算结果（换仓/注资两种模式）

期权

  • 持仓指标：VGT/SMH 持仓市值、成本、可卖张数 + 可复投金额
  • 🎯 OTM 行权价参考：VGT/SMH 按百分比自动计算 CALL 行权价，可手动调节后 localStorage 记忆
  • 📝 记录期权：录入 CALL/PUT 合约（标的、类型、行权价、权利金、到期日、张数）
  • 期权状态：当前活跃期权的盈亏状态 — 实值（ITM）CALL 显示蓝色「临近行权」标签（显示剩余天数），附带铁律提示文字
  • CC 收入：本月 + 累计权利金汇总 + 可复投金额
  • 持仓清单：当前持有合约（含张数、到期状态、实值/虚值标记）

📁 数据

  • 💰 现金管理：入金/出金/修正 + 余额
  • 📊 持仓明细：标的/持仓/均价/现价/市值/盈亏，含张数列，支持列排序
  • 📋 交易历史：全部记录 + 标的筛选 + 列排序
  • 💵 资金流水：出入金记录 + 修正记录 + 权利金蓝色独立显示

📜 日志·规划

  • 📊 年度复盘矩阵：4行×5列热力方格，年度权利金贡献率（CC权利金÷总买入）驱动着色；每格显示年份/贡献率/年末持仓（VGT/SMH/BTC股数）/CC权利金 | 总买入；右侧统计面板含总入金、总资产、CAGR、目标差额；移动端自动切换2列布局
  • 🔥 纪律打卡：连续定投月份计数 + 操作日志热力图
  • 📈 收入渐进加码：四阶段年龄自动计算（起步期/增长期/巅峰期/冲刺期）
  • 🚪 退出策略：47岁按回撤切换防守组合（可自定义保守组合比例）
  • 💰 提款模拟：输入资产/提取率/回报/通胀，计算年/月提取额、耗尽时间、永续率

2. 侧边栏

  • 总资产：总额/持仓/现金 + 持仓环形图
  • 实时价格：VGT/SMH/BTC/SGOV 实时行情（来源标注）
  • ⚙️ 投资参数：每月定投总额、定投起点日期、起点年龄、财务目标、仓位分配滑块
  • 🎨 配色方案：5 种主题色（森林/海洋/暖阳/梅紫/极简），切换后记忆偏好
  • ☁️ 数据同步：Cloudflare D1 云端 + JSON 导出 + CSV 导入 + 自动备份
  • 🌙/☀ 暗色/亮色模式：首次自动跟随系统设置，手动切换后记忆偏好

3. 云端同步

侧边栏底部「☁️ 云端同步」区域：

  • 自动上传：录入交易、入金出金后 2 秒自动推云端
  • 自动下载：页面打开时从云端拉最新数据合并
  • 状态指示：数据区灰点（上传中→橙，成功→绿，失败→红）；右下角气泡提示
  • 线上版自动填 URL，Token 需在侧栏同步面板手动填入（与 Worker 环境变量 AUTH_TOKEN 一致）
⚠ 多设备冲突：两设备同时编辑分别上传，后者覆盖前者。建议一台编辑完等同步完成再换设备。

4. Schwab CSV 导入

  • 拖拽：CSV 文件拖到页面任意位置（显示拖放覆盖层）
  • 按钮：侧边栏「CSV」按钮选择文件
  • 支持买入和卖出记录自动识别
  • 导入 VGT / SMH / BTC

5. 数据备份

  • 「📤 导出」→ 下载 JSON 文件（交易 + 资金 + 参数 + 期权 + 价格），Toast 提示数量
  • 「📥 导入」→ 选择 JSON 文件恢复，Toast 提示数量
  • 勾选「自动备份」→ 每次操作自动下载，显示上次备份时间

6. 现金修正

  • 数据页现金管理区域输入金额
  • 正数增加现金，负数减少现金
  • 自动记录到资金流水（修正+/修正-）
  • 适用于网站记录与实际账户不同步时的手动校准

7. Toast 提示

所有操作反馈通过底部居中 Toast 气泡展示，统一毛玻璃效果 + 弹性动画：

  • 成功（绿色）：半透明绿底 + 绿色阴影，用于保存/同步/导出等成功操作
  • 错误（红色）：半透明红底 + 红色阴影，用于失败/解析错误
  • 警告（橙色）：半透明橙底 + 橙色阴影，用于提醒/注意
  • 撤销（下划线）：深色底 + 下划线，点击可撤销刚才的操作（如删除交易/清空流水）

所有 Toast 统一毛玻璃效果（backdrop-filter: blur(28px) saturate(180%)），弹性弹入动画（cubic-bezier(.34,1.56,.64,1)），自动 2.5-5 秒后消失。

8. 表格滚动与表头固定

持仓明细、交易历史、资金流水、期权清单等表格的 表头 已设置 sticky 定位。当表格内容超出卡片高度滚动时，表头自动固定在顶部，始终保持可见，方便对照列名查看数据。

9. 期权模块 · 被动行权 T+1

期权模块采用被动行权 T+1 买回策略（策略建议仅卖 CALL，但类型选择器保留 CALL/PUT 选项）：

  • 实值（ITM）CALL：不再显示红色告警，改为蓝色「临近行权」标签（显示剩余天数），同时显示铁律提示文字
  • 铁律卡片：期权页顶部显示绿色弱底提示「切勿主动平仓，被行权不必慌，T+1 市价买回，踏空代价远高于追高成本」
  • 被动行权：若到期时 CALL 处于实值被行权，在 T+1 手动买回正股，不提前平仓
  • 已结算：被行权后 CALL 状态变为「已结算」褪色显示
  • 权利金流程：录入 CALL/PUT 时权利金实时入账现金，不计入入金/出金，独立追踪

10. 资金流水 · 权利金独立

  • 权利金：录入期权时自动创建资金流水记录，类型为「期权权利金」
  • 独立显示：权利金以蓝色标记，不计入入金（绿色）也不计入出金（红色）
  • 账户现金卡片：顶部「入金」显示排除权利金，仅统计 cashLog 中入金类型
  • 合计归属：资金流水合计中权利金归入流入侧

11. 纪律打卡与投资规划

  • 连续定投：日志页显示连续定投月份计数
  • 操作日志热力图：按日可视化操作频率
  • 收入渐进加码：根据起点年龄自动划分四阶段（起步/增长/巅峰/冲刺）
  • 退出策略：47岁按回撤切换防守组合，可自定义保守组合比例
  • 提款模拟：输入资产/提取率/回报/通胀，计算提取额、耗尽时间和永续率

12. 年度复盘矩阵

日志页纪律打卡卡片下方展示 20 年年度复盘热力矩阵：

  • 热力着色：每格背景色由「年度权利金贡献率」决定（CC权利金÷总买入），颜色越暖权利金贡献越大
  • 每格信息：年份 → 贡献率（加粗大字）→ 年末持仓（VGT/SMH/BTC 股数）→ CC权利金 | 总买入
  • 右侧长线数据：总入金 / 当前总资产 / 长期CAGR / 目标差额
  • 年份起点：从首笔买入或入金记录的实际年份开始，最多 20 年
  • 移动端：自动切换 2 列网格 + 统计面板 2×2 紧凑排列
  • 热力分段：≥12% 深红 / ≥10% 橙红 / ≥8% 浅橙 / ≥6% 米黄 / ≥3% 浅青 / 0.1%+ 浅蓝 / 0% 深蓝

13. 移动端

  • 手机浏览器直接打开网址
  • 底部导航 5 个 Tab 切换页面
  • 顶部 sticky 状态栏始终保持可见，显示总资产 + 实时价格
  • 期权 Tab 图标为底仓+CALL（box + up-arrow 组合图标）
  • 侧边栏点左上 ☰ 展开，带遮罩层
  • 表单和按钮最小 44px 点击区域
  • 顶部状态栏点击可下拉刷新价格

14. 暗色模式与配色

首次打开自动跟随系统设置。手动切换后记忆偏好，不再跟随系统。侧边栏点 🌙/☀ 切换暗色/亮色模式。5 种配色方案可选（森林/海洋/暖阳/梅紫/极简），切换后 localStorage 记忆。暗色模式下卡片有明确边界。

15. UI 交互细节

  • 按钮体系统一：所有按钮带阴影、悬浮过渡、按压缩放（:active → scale(.97)）
  • 主要按钮：绿色填充 + 绿色阴影，悬浮时加深 + 上浮 1px
  • 次要按钮：灰色表面 + 边框，悬浮时上浮
  • 卡片系统统一：统一圆角 + 投影，悬浮时阴影加深 + 上浮 1px
  • Tab 切换反馈：切换 Tab 时移动端触发 8ms 触觉震动
  • 输入框：focus 时显示绿色光环
  • 持仓分配滑块：侧边栏投资参数区可通过滑块调节 VGT/SMH/BTC 定投分配比例
